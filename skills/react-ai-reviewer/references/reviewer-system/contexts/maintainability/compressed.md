# Maintainability 압축 컨텍스트

- 같은 이유로 바뀌는 코드는 모여 있어야 하고, 다른 이유로 바뀌는 코드는 분리되어야 한다.
- UI, validation, mapping, analytics, fetch orchestration이 한 함수에 섞이면 변경 비용이 급격히 커진다.
- 조건 분기와 숨은 정책이 여러 곳에 퍼지면 다음 요구사항에서 수정 범위가 넓어진다.
- 추상화는 함수 수를 늘리기 위한 수단이 아니라 변경 흐름을 단순하게 만들기 위한 수단이어야 한다.
- 작은 PR에는 과한 분해보다 수정 경로를 짧게 만드는 정리가 더 낫다.
